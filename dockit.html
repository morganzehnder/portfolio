THE GIST

THE GIST

Summary

There are various tools on the web for saving content. Users find themselves saving and sorting content in various ways dependent on the nature of the content and functions of specific tools. Dockit is a platform built for individuals and teams to store and organize all types of web content, with the ability to create or upload supplemental content.

Problem

Users need a single platform fit to store all types of content for ease of organization and file relocation.

Solution

Design a platform fit to store and showcase all types of content so users have all of their files in a single location.

Competitive Analysis

File storage web applications offer different sets of tools for perfoming relatively the same task - saving content. Before deciding on the features to include in Dockit, I conducted a competitive analysis to assess the strengths and weaknesses of competitors against our users needs.

PRO

Quick and easy for saving visual content

CON

Not good for any non-visual content

PRO

Ability to clip content, don't have to save whole web page

CON

More complicated interface & org. system, focus on text-based, educational material

PRO

Ability to tag right when you save, easy organization

CON

Cannot upload or create supplemental content

PRO

Easy to collaborate and share files

CON

No way to save files directly from web

Navigation

The site structure of Dockit is based on the hub and spoke model, with the Dashboard serving as the hub. Based on my research, users need a simple and flexible organization system that allows them to make the organizational choices. I've opted to utilize tags as the organizational system for this reason.

As a result, most actions can be taken directly in the dashboard. The inclusion of a collaborator and activity box right on the main page of the dashboard allows users to search/filter by tag right within the dash and also see all the collaborators and activity for that specific group of files. As they filter by tags, the "Collab Box" will update for that specific group of files. As such, users can also search by tag and simply add a collaborator to that entire group of files easily.

I prioritized building an extremely simple, yet flexible system. Users don't need a ton of bells and whistles, just a single system that allows them to easily add and relocate all types of files, add collaborators, view activity, and define their own organizational system.

Color

As a tool users would use frequently, I wanted to keep color choices engaging, yet simple and clear. Using a light gray as the base color, I delineated between sections on the dashboard utilizing different action colors. Utilizing green for account settings, orange for collaboration, and a light red for content. Using color as code further delineates areas while adding a playful quality.

TESTING

TESTING

A / B Test

Adding your own images, notes, and links is an important feature on Dockit. I conducted a preference test to see where the “add content” button should be placed on the dashboard to make it as clear and convenient as possible to users.

Results

The second option was clearer and easier to access for every tester but one.

Takeaway

The large, round, red button laying within the content section itself more clearly indicates to users where to complete the task.

Navigation Test

I conducted two navigation tests to confirm the large, red add content button was the correct navigation choice. I conducted a second navigation test because my first was skewed as a result of one user navigating away from their screen, leaving the time running. 95% of users successfully completed the task, but it took an average of 9 seconds.

Results

On the second test, with new, random users, 100% of users completed the test successfully in 6 seconds time.

Takeaway

Moved forward with the second option in my design.

5 Second Test

I conducted a five second test of my homepage to ensure my landing conveys it’s brand and message quickly to users.

Results

  • 80% of testers believed the page looked trustworthy
  • Almost every user noticed a different item on the homepage
  • 90% of users understood the nature of the site

Takeaway

Decide what is most important on the homepage, simplify, and make it stand out.

What I Learned

For this project, there were lots of iterations along the way in terms of branding. Working this project from start to finish, branding the site was very important to me which led to several iterations of my project, especially logo design. Rebranding was both a blessing and a curse. The future iterations proved to be improvements and brought a sense of cohesiveness to the site. However, several rounds of iterations slowed me down a bit. As this was my first project, I am still working out my design process to make it the most effective and efficient. In the future, if I am tasked with branding a site, I will make sure to have fully developed branding choices that I am confident with before moving forward with design. It can definitely can be a challenge to work in reverse.

Throughout user testing the site received lots of positive feedback and users were able to easily complete the tasks required of them. They liked the simple interface and understood what the site had to offer. I relied on simplicity and narrowing down the number of features to the ones users absolutely wanted to enable them to use a single site to store their content. These features included: bookmarking browser application, ability to create notes and comment, a platform that showcases all types of content, and the ability to add web and non-web based content. I relied heavily on early research and worked to give users all the key features they desired from the start. This is definitely a quality I will carry forward into future projects. It served as a solid foundation and delivered my end goal.
